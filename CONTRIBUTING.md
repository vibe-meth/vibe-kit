## Submitting Changes

### Before You Start

1. Open an issue or discussion first
2. Wait for feedback
3. Get approval for major changes

### Making Your Change

1. Create a feature branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test thoroughly
4. Commit with clear messages: `git commit -m "Add clarify command"`
5. Push to your fork: `git push origin feature/your-feature`
6. Open a Pull Request

### PR Guidelines

- **Title**: Clear and descriptive
- **Description**: Explain what and why
- **Testing**: Note how you tested
- **Docs**: Update if needed
- **One change per PR**: Easier to review

## Code Style

- **Python**: Follow PEP 8
- **Shell scripts**: Use `shellcheck`
- **Documentation**: Clear, concise, examples
